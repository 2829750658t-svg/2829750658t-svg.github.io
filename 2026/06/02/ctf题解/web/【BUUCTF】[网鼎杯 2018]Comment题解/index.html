知识点：

1.修复git源码git reset --hard

传入指定的哈希值后，Git 会在底层同时修改三个区域的状态：

HEAD 指针⟶暂存区 (Index)⟶工作区 (Working Directory)

底层操作链：
  1. 移动指针：将当前分支（master）和 HEAD 指针强行移动到 bfbdf218902476c5c6164beedd8d2fcf593ea23b 这个 Commit 对象上。
  2. 重置暂存区 (Index)：清空当前的暂存区，将该 Commit 对象中记录的 Tree 对象（目录树快照）内容写入暂存区。
  3. 覆盖工作区 (Working Directory)：（关键） Git 检查你当前操作系统下的物理磁盘文件夹（即 D:\official\githacker\...），删除不属于该版本的所有文件，并根据 Tree 对象中的 Blob（文件内容）在磁盘中真实解压并还原出所有的 PHP 源码文件。
